# instructions.md  
  
## Purpose of This File  
  
This document gives Claude Code the framing, initial direction, and approach for building my personalized learning roadmap for **agentic web design**. It is the starting point — not a fixed specification. The roadmap should evolve as I progress and learn more about what works for me.  
  
-----  
  
## Context About Me  
  
- I want to get genuinely good at agentic web design (using AI agents like Claude Code to build high-quality web UIs).  
- I am **not** aiming to become a traditional full-stack developer. My focus is on directing, critiquing, and iterating with AI — not memorizing syntax.  
- I learn best by **doing, breaking, fixing, and iterating** — not by passive reading.  
- I want progress to feel tangible. I need concrete deliverables, feedback mechanisms, and checkpoints.  
  
-----  
  
## Initial Roadmap Skeleton (Guiding Direction, Not Gospel)  
  
The roadmap should build on the following 6-stage skeleton, which emerged from earlier thinking. Treat this as a **starting frame**. Feel free to restructure, merge, expand, or re-sequence stages based on what the research supports — but preserve the spirit of progressive, doing-first learning.  
  
### Stage 1 — Get Hands Dirty  
  
Build something ugly that works. Use v0, Bolt, or Claude Code to create a personal portfolio. Read every line of generated output. Break things intentionally.  
  
### Stage 2 — Develop Critique Skills  
  
Learn to see. Compare output to sites I admire. Enumerate specific differences. Re-prompt with those critiques. Build an eye before building complexity.  
  
### Stage 3 — Build in Components  
  
Learn to scope prompts properly. Build a multi-section landing page in isolated pieces (navbar, hero, features, footer) and combine them.  
  
### Stage 4 — Add Interactivity  
  
Move from static to dynamic. Introduce a real interactive feature (form, toggle, filter). Force engagement with JS/React basics for debugging.  
  
### Stage 5 — Clone Something Real  
  
Close the gap between taste and output. Clone a site I admire from memory via prompting. Iterate until close.  
  
### Stage 6 — Build for a Real Problem  
  
Build something I’d actually use or show. Constraints now matter: responsiveness, accessibility, performance. Audit with Lighthouse.  
  
**The loop that runs through every stage:**  
`Prompt → Study output → Break something → Fix it → Prompt better`

## How Claude Code Should Approach the Research Docs  
  
There are research documents in the `/research/` folder produced via Claude Deep Research. They cover two focus areas:  
  
1. **Learning science for creative/judgment-based skills** — how deliberate practice, feedback loops, and taste development work for skills that aren’t purely technical.  
1. **Self-directed learning plan design** — task sizing, pacing, checkpoints, plateau navigation.  
  
There may also be research on:  
  
- Current agentic tooling landscape (v0, Bolt, Lovable, Claude Code, etc.)  
- Best learning resources for Tailwind + React  
- Common mistakes in agentic frontend development  
- Claude Code design skills (Impeccable, frontend-design)  
  
### Rules for Using the Research  
  
- **Read all files in `/research/` before drafting the roadmap.** Don’t skim.  
- **Integrate the learning science deeply.** The final roadmap should reflect:  
  - Appropriate task sizing (not too easy, not overwhelming)  
  - Built-in feedback loops at every stage  
  - Explicit checkpoints before progression  
  - Strategies for plateau navigation  
  - Deliberate practice principles adapted for taste/judgment skills  
- **Don’t just list resources.** Curate. If research mentions 10 options, recommend 1-2 and explain why.  
- **Flag conflicts.** If research contradicts the initial skeleton, surface that explicitly and suggest an alternative.  
- **Cite research inline** when making structural choices (e.g., “Based on research on desirable difficulty, this stage includes…”).

## What I Want the Final Roadmap to Contain  
  
Every stage should include:  
  
- **Goal** — what I’ll be able to do at the end  
- **Why this stage matters** — the rationale in terms of skill acquisition  
- **Specific projects/deliverables** — concrete, doable, shippable  
- **Practice mechanics** — how to structure the work (session length, iteration cadence, etc.)  
- **Feedback mechanism** — how I’ll know I’m improving (self-critique, community, metrics)  
- **Checkpoint** — criteria to meet before moving on  
- **Estimated time** — realistic, with range rather than fixed  
- **Plateau indicators** — signs I’m stuck and what to try  
  
At the top, include:  
  
- A clear philosophy section  
- An overview of the full arc  
- A “how to use this roadmap” note  
  
At the end, include:  
  
- What happens after Stage 6  
- How and when to revise this roadmap  
- A personal tracking section (what I completed, what I learned)
