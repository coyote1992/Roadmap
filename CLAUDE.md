# CLAUDE.md  
  
## Project Purpose  
  
This repository is the home of my self-directed learning journey into **agentic web design** — using AI agents (primarily Claude Code) to build and iterate on web interfaces with professional quality.  
  
The goal is not to become a full-stack programmer. The goal is to become excellent at directing AI agents to produce high-quality frontend work, which requires:  
  
- Enough technical fundamentals to critique and fix AI output  
- Strong design taste and vocabulary  
- Mastery of iterative prompting and refinement  
- Judgment about tooling, workflows, and quality standards  
  
## Learning Philosophy  
  
- **Learn by doing, not by reading.** Every concept is reinforced through a concrete project or deliverable.  
- **Iterate relentlessly.** Generate → critique → refine → repeat. This loop runs through every stage.  
- **Build taste before borrowing skills.** Understand why something works before installing someone else’s shortcut.  
- **Scope tightly.** Tasks should be small enough to complete in a session, big enough to stretch capability.  
  
## Current Stack  
  
- **Primary agent:** Claude Code (in VS Code)  
- **Styling:** Tailwind CSS  
- **Framework:** React  
- **Design skills of interest:** Anthropic’s `frontend-design`, Impeccable by Paul Bakaus  
- **Research tool:** Claude Deep Research (outputs stored in `/research`)  
  
## Repo Structure  
  
- `instructions.md` — Initial direction and framing for the roadmap. Read this first.  
- `roadmap.md` — The living learning roadmap (to be created).  
- `/research/` — Deep Research outputs that inform the roadmap.  
- `/projects/` — Code for each stage’s deliverables.  
- `/notes/` — Personal reflections, critiques, and lessons learned per stage.  
  
## How Claude Code Should Behave  
  
### Every Session  
  
1. Read `instructions.md` for framing.  
1. Read `roadmap.md` to understand where I am.  
1. Skim `/research/` files if the task involves planning, restructuring, or evaluating the roadmap.  
  
### When Building or Updating the Roadmap  
  
- Treat `/research/` as the authoritative source of learning-science and tooling context. Integrate it, don’t ignore it.  
- Preserve my current stage and progress. Don’t rewrite history.  
- Favor concrete, doable tasks over abstract advice.  
- Every stage must have: a goal, specific deliverable(s), feedback mechanism, and a checkpoint before moving on.  
  
### When Helping With Projects  
  
- Prefer Tailwind + React unless I say otherwise.  
- Show me *why*, not just *what*. Explain tradeoffs.  
- When I ask for code, also flag what I should critique in the output.  
- If I’m making a mistake a more experienced developer would catch, tell me.  
  
### Tone  
  
- Direct, honest, no filler.  
- Push back when my thinking is off.  
- Don’t flatter. Don’t pad.  
  
## Non-Negotiables  
  
- Never reframe this as a “learn to code from scratch” journey. This is agentic generation, which is a different discipline.  
- Never strip learning-science principles out of the roadmap for the sake of simplicity.  
- Never recommend skills, tools, or shortcuts I’m not yet ready to evaluate critically.
